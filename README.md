# 人间剧场 🎭

**AI驱动的多智能体叙事模拟平台** —— 不写剧本，只定规则，让角色在设定中自主成长、演绎无法预测的故事。

## 核心理念

> 每个角色都拥有独立的灵魂。

传统叙事工具需要你写好每一句对白、每一个情节转折。人间剧场不同——你只需要定义角色、关系和场景，AI 会为每个角色注入独立的思维，让他们在规则内自由互动，生成独一无二的剧情。

- **小说体** 📖 — AI 以文学叙事风格生成连贯章节（500-800字/章）
- **多智能体** 🎬 — 每个角色独立"思考"，产生内心想法、动作和对话，由导演LLM编排为连贯场景

## 功能特性

- **角色管理** — 自定义角色的姓名、性别、年龄、外貌、性格、背景故事
- **关系设定** — 定义角色之间的双向关系（每个角色对同一段关系可以有不同认知）
- **场景设定** — 自由配置时间和地点，为故事搭建舞台
- **剧场创建** — 将角色、关系、场景组合为一个剧场配置
- **双模式演绎** — 小说体 / 多智能体两种叙事模式
- **实时干预** — 在任意时刻输入导演指示，引导剧情走向
- **分层摘要压缩** — 自动压缩过长上下文，保持长故事连贯性
- **存档管理** — 保存/继续剧场进度，支持导出TXT

## 快速开始

### 前置条件

- Node.js >= 18
- 一个兼容 OpenAI 接口的大模型 API 密钥（支持 OpenAI / DeepSeek 等）

### 安装与运行

```bash
# 1. 安装前端依赖
npm install

# 2. 安装后端依赖
cd server && npm install && cd ..

# 3. 同时启动前后端（开发模式）
npm run dev
```

前端开发服务器运行在 `http://localhost:5173`，后端 API 服务器运行在 `http://localhost:3001`。

### 生产构建

```bash
# 构建前端
npm run build

# 构建后端
npm run build:server

# 启动生产服务
npm start
```

## 多智能体引擎

这是项目的核心技术。采用**两阶段架构**：

1. **Phase 1: 角色思考** — 为每个角色独立调用 LLM，让角色以第一人称思考当前情境（内心想法、动作、对话、目标、情绪）。角色分批并行调用（每批3个），单角色失败不阻塞整体。

2. **Phase 2: 场景编排** — 导演 LLM 接收所有角色的思考结果，整合为连贯的 JSON 场景，确保关系一致性、知识边界、全员出场。

**关键技术**：Fisher-Yates 角色洗牌消除首位偏差、关系矩阵约束知识边界、三层 JSON 容错解析、多层降级策略。

## 使用流程

1. **注册账号** — 填写用户名、密码和大模型 API 密钥
2. **创建角色** — 定义角色的外貌、性格、背景故事等
3. **设定关系**（可选）— 建立角色之间的双向关系认知
4. **选择场景** — 设定故事的时间和地点
5. **开始剧场** — 选择小说体或多智能体模式，AI 开始演绎
6. **继续/干预** — 持续生成剧情，或随时输入指令改变方向
7. **存档/导出** — 保存进度，或导出为 TXT 文件
